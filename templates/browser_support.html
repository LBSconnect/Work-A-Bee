Browser Support Guide

Welcome to the Work-A-Beez Browser Support Guide.

Work-A-Beez is a modern, cloud-native web application designed to operate using current versions of major web browsers. This guide explains supported browsers, browser configuration requirements, compatibility policies, and troubleshooting recommendations.

Following these guidelines helps ensure optimal performance, security, and user experience.

Overview

Work-A-Beez is accessed using a modern web browser over a secure HTTPS connection.

No browser plug-ins or desktop software are required.

For the best experience:

  • Keep browsers updated.
  • Enable JavaScript.
  • Enable cookies.
  • Use supported browser versions.
  • Avoid unsupported browser extensions.

Browser Support Policy

Work-A-Beez supports current vendor-supported releases of major browsers.

As browser vendors release new versions, Work-A-Beez regularly validates compatibility.

Support generally includes:

  • Current Stable Release
  • Previous Stable Release (when still vendor-supported)

Browsers that have reached end-of-life or no longer receive security updates are not supported.

Supported Desktop Browsers

Google Chrome

Supported:

  • Current stable release
  • Previous stable release (while vendor-supported)

Microsoft Edge

Supported:

  • Current stable release
  • Previous stable release

Mozilla Firefox

Supported:

  • Current Extended Support Release (ESR)
  • Current stable release

Apple Safari

Supported on current vendor-supported versions of macOS.

Safari users should ensure:

  • Cookies are enabled
  • JavaScript is enabled
  • Pop-up blocking allows approved Work-A-Beez pages when required

Supported Mobile Browsers

Work-A-Beez does not currently offer a dedicated mobile application; the web application is responsive and supports use from mobile browsers, including:

iOS

  • Safari
  • Chrome (current supported version)

Android

  • Google Chrome
  • Firefox

Browser Configuration Requirements

The following browser features must be enabled.

HTTPS

All connections use encrypted HTTPS.

HTTP connections are not supported.

TLS

Browsers must support:

  • TLS 1.2 or higher

Older SSL and TLS protocols are not supported.

Pop-ups

Some administrative features may open secondary browser windows.

Organizations should allow pop-ups for approved Work-A-Beez domains where necessary.

Downloads

Browsers should allow downloads where reports or documents are provided.

Cookies & Local Storage

Work-A-Beez requires cookies for:

  • Authentication
  • Session management
  • Security

Blocking required cookies may prevent successful sign-in or normal application behavior.

JavaScript Requirements

JavaScript is required for nearly all platform functionality.

Disabling JavaScript may prevent:

  • Authentication
  • Navigation
  • Dashboards
  • Scheduling
  • Reporting
  • Forms

Browsers should permit JavaScript for all Work-A-Beez domains.

Browser Security Requirements

For security reasons, browsers should:

  • Receive automatic updates
  • Support modern encryption
  • Validate digital certificates
  • Warn users of insecure websites
  • Support secure cookies

Users should avoid disabling browser security features.

Browser Extensions

Most browser extensions are compatible with Work-A-Beez.

However, some extensions may interfere with normal operation.

Examples include:

  • Aggressive content blockers
  • Script-blocking extensions
  • Cookie removal tools
  • Privacy extensions that disable required storage
  • Browser automation tools

If unexpected behavior occurs, temporarily disable browser extensions to determine whether they are causing the issue.

Browser Compatibility Testing

Work-A-Beez regularly tests major application functionality, including:

  • Authentication
  • Dashboard navigation
  • Employee management
  • Scheduling
  • Timekeeping
  • Leave management
  • Payroll preparation
  • Reporting
  • Administrative configuration
  • Mobile responsiveness

Testing occurs throughout the software development lifecycle.

Unsupported Browsers

The following browser configurations are not supported:

  • Internet Explorer
  • Browsers no longer maintained by their vendor
  • Browsers with JavaScript disabled
  • Browsers with required cookies disabled
  • Browsers using obsolete TLS versions
  • Experimental browser builds not intended for production use

Unsupported browsers may experience reduced functionality or security vulnerabilities.

Known Limitations

Certain browser-specific behaviors may differ slightly.

Examples include:

  • Native file selection dialogs
  • PDF viewing behavior
  • Notification permissions
  • Password manager integration
  • Printing interfaces

These differences are determined by the browser vendor rather than the Work-A-Beez application.

Browser Troubleshooting

Application Will Not Load

Verify:

  • Internet connectivity
  • Browser version
  • JavaScript enabled
  • HTTPS access
  • Firewall settings

Unable to Sign In

Check:

  • Cookies enabled
  • Correct credentials
  • Supported browser version

Slow Performance

Recommendations:

  • Close unused browser tabs
  • Disable unnecessary browser extensions
  • Restart the browser
  • Clear browser cache
  • Update to the latest browser version

Display Issues

Try:

  • Refresh the page
  • Clear cached files
  • Zoom level set to 100%
  • Disable browser compatibility mode
  • Update graphics drivers if necessary

Frequently Asked Questions

Which browser is recommended?

Google Chrome and Microsoft Edge generally provide the best overall experience.

Can I use Safari?

Yes.

Current supported versions of Safari are fully supported on macOS and iOS.

Does Work-A-Beez support Internet Explorer?

No.

Internet Explorer is not supported.

Do I need browser plug-ins?

No.

Work-A-Beez does not require browser plug-ins or add-ons.

Should browsers be updated automatically?

Yes.

Automatic browser updates improve both security and compatibility.
