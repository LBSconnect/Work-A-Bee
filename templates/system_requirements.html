Introduction

Work-A-Beez is a cloud-native Software-as-a-Service (SaaS) platform.

Because the application is browser-based, customers do not install application servers or desktop software. Users access Work-A-Beez through a supported web browser.

Supported Deployment Model

Work-A-Beez supports:

  • Cloud-hosted SaaS deployment
  • Multi-tenant architecture
  • Secure HTTPS access

No on-premises application installation is required.

Hardware Requirements

Minimum Workstation

  • Dual-core processor
  • 4 GB RAM
  • 2 GB available disk space
  • Internet connection
  • 1366 x 768 display resolution

Recommended Workstation

  • Quad-core processor or better
  • 8 GB RAM or more
  • SSD storage
  • Stable broadband Internet
  • Full HD (1920 x 1080) display or higher

Administrator Workstations

For administrators who frequently manage reports, imports, and payroll processing:

Recommended:

  • Quad-core processor
  • 16 GB RAM
  • Dual monitors (recommended)
  • High-speed Internet connection

Operating System Requirements

Supported operating systems include current vendor-supported versions of:

Microsoft Windows

  • Windows 10
  • Windows 11

Apple macOS

Current supported versions of macOS.

ChromeOS

Supported through compatible web browsers.

Linux

Current enterprise Linux distributions running supported browsers.

Operating systems that are no longer supported by their vendors are not recommended.

Browser Requirements

Work-A-Beez supports current stable versions of major browsers including:

  • Google Chrome
  • Microsoft Edge
  • Mozilla Firefox
  • Apple Safari

Browsers should:

  • Support JavaScript
  • Support cookies
  • Support TLS 1.2 or higher

See the Browser Support Guide for detailed compatibility information.

Mobile Device Requirements

Work-A-Beez does not currently offer a dedicated mobile application. The web application is responsive and supports use from mobile browsers, including:

Apple iOS

Current supported versions of iOS, using Safari.

Android

Current supported versions of Android, using Chrome.

Some features are optimized for larger screens and may be more limited on small mobile displays.

Display Requirements

Minimum supported resolution:

  • 1366 x 768

Recommended resolution:

  • 1920 x 1080 or higher

The user interface is responsive and adapts to tablets, laptops, and desktops.

Network Requirements

Organizations should ensure access to:

  • HTTPS (TCP Port 443)
  • DNS resolution
  • Secure outbound Internet access

Recommended network characteristics:

  • Low latency
  • Stable connectivity
  • Reliable DNS services

If outbound traffic is filtered, Work-A-Beez domains should be added to the organization's allowlist.

Internet Connectivity

Recommended connection:

  • Broadband Internet

Suggested minimum bandwidth:

  • 5 Mbps per active workstation

Higher bandwidth may be beneficial for simultaneous users and larger organizations.

Email Requirements

Organizations should permit email delivery from approved Work-A-Beez sending domains.

Examples include:

  • Account activation
  • Password resets
  • Schedule notifications
  • Weekly payroll reports
  • System alerts

Administrators should review spam filtering policies to ensure delivery of transactional emails.

Authentication Requirements

Supported authentication method:

  • Username/email and password

Additional authentication methods (Multi-Factor Authentication, Single Sign-On) are planned but not yet available.

Printing Requirements

For organizations printing schedules or reports:

Recommended:

  • Standard office printer
  • A4 or Letter paper support

Reports are currently viewed on-screen through the browser; browser print/PDF functions can be used to print or save a copy.

Performance Recommendations

For the best experience:

  • Use the latest browser version.
  • Enable automatic browser updates.
  • Close unused browser tabs.
  • Use wired or high-quality Wi-Fi connections.

Organizations should periodically review browser and operating system versions to maintain compatibility.

Unsupported Configurations

The following configurations are not supported:

  • Unsupported or end-of-life operating systems
  • Browsers that no longer receive vendor security updates
  • Browsers with JavaScript disabled
  • Browsers with required cookies disabled
  • Insecure HTTP connections

Unsupported environments may experience reduced functionality or security risks.

Frequently Asked Questions

Do I need to install software?

No.

Work-A-Beez is accessed through a supported web browser.

Can I use Work-A-Beez on a tablet?

Yes.

The responsive interface supports most modern tablets using supported browsers.

Is VPN access required?

Typically, no.

Organizations with specific security policies may require VPN access for certain users.

Does Work-A-Beez support dual monitors?

Yes.

Dual monitors are particularly useful for payroll administrators, schedulers, and HR personnel managing large datasets.

Will Work-A-Beez work on older computers?

In many cases, yes.

However, modern hardware and current operating systems provide the best user experience and security.
