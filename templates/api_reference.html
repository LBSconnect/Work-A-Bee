Authentication Endpoints

Authentication endpoints allow applications and users to securely obtain and manage access tokens.

Authenticate User

Creates an authenticated session and returns an access token.

Endpoint

http POST /v1/auth/login

Authentication

Not required.

Request Headers

http Content-Type: application/json

Request Body

json { "email": "john.doe@example.com", "password": "StrongPassword123!" }

Successful Response

http 200 OK

json { "success": true, "accessToken": "eyJhbGciOi...", "refreshToken": "eyJhbGciOi...", "expiresIn": 3600, "tokenType": "Bearer" }

Possible Errors

Status Description
400 Malformed request
401 Invalid credentials
423 Account locked
429 Too many login attempts

Refresh Access Token

Obtains a new access token using a valid refresh token.

Endpoint

http POST /v1/auth/refresh

Request

json { "refreshToken":"eyJhbGc..." }

Successful Response

json { "success": true, "accessToken":"eyJhbGc...", "expiresIn":3600 }

Logout

Terminates the current authenticated session.

Endpoint

http POST /v1/auth/logout

Authentication

Bearer Token Required

Response

http 204 No Content

Employee Endpoints

Employee endpoints allow administrators and authorized applications to manage employee records.

List Employees

Returns a paginated collection of employees.

Endpoint

http GET /v1/employees

Required Scope

Employee.Read

Query Parameters

Parameter Description
page Current page number
pageSize Results per page
department Filter by department
location Filter by location
status Filter by status
managerId Filter by manager

Example Request

http GET /v1/employees?page=1&pageSize=25

Example Response

json { "success": true, "data":[ { "employeeId":"EMP-10025", "firstName":"John", "lastName":"Smith", "department":"Operations", "jobTitle":"Supervisor", "status":"Active" } ], "pagination":{ "page":1, "pageSize":25, "totalRecords":684, "totalPages":28 } }

Retrieve Employee

Returns detailed information about a single employee.

Endpoint

http GET /v1/employees/{employeeId}

Path Parameters

Parameter Description
employeeId Unique employee identifier

Successful Response

json { "success":true, "data":{ "employeeId":"EMP-10025", "firstName":"John", "lastName":"Smith", "email":"john.smith@example.com", "department":"Operations", "location":"Houston", "manager":"Jane Doe", "hireDate":"2025-01-12", "status":"Active" } }

Create Employee

Creates a new employee record.

Endpoint

http POST /v1/employees

Required Scope

Employee.Write

Request Body

json { "firstName":"John", "lastName":"Smith", "email":"john.smith@example.com", "departmentId":"OPS", "locationId":"HOU", "jobTitleId":"SUP" }

Successful Response

http 201 Created

Returns the complete employee object including its newly assigned Employee ID.

Update Employee

Updates an existing employee record.

Endpoint

http PUT /v1/employees/{employeeId}

Only fields supplied in the request body are modified.

Delete Employee

Marks an employee record as inactive.

Endpoint

http DELETE /v1/employees/{employeeId}

Employee records are retained for audit and reporting purposes. This endpoint performs a logical deletion rather than permanently removing historical records.

Standard Error Codes

All endpoints may return standardized error responses.

HTTP Status Description
200 Request completed successfully
201 Resource created
204 No content
400 Bad request
401 Authentication required
403 Permission denied
404 Resource not found
409 Conflict
422 Validation failed
429 Rate limit exceeded
500 Internal server error
503 Service temporarily unavailable

Standard Error Response

json { "success": false, "error": { "code":"ValidationError", "message":"One or more fields failed validation.", "details":[ { "field":"email", "message":"Email address already exists." } ], "traceId":"6fb3411d-3d14-44d8-8e78-acde12345678" } }

The traceId should be included when contacting Work-A-Beez Support to help identify the associated request in server logs.

API Conventions

All endpoints follow these standards:

  • HTTPS only
  • UTF-8 encoding
  • JSON request and response bodies
  • ISO 8601 date and time format
  • CamelCase property names
  • Consistent pagination model
  • Standardized error handling
  • Audit logging for all write operations
  • Role-based authorization checks
