Work-A-Beez Security Overview

Last updated: {{ updated.strftime('%B %d, %Y') }}

⚠ Multi-Factor Authentication (MFA) described in this document is a planned feature and is not yet available on Work-A-Beez.

At Work-A-Beez, security is a core part of how we design, build, and operate our platform.

We understand that our customers entrust us with sensitive workforce information, and we are committed to protecting that information through industry-recognized security practices, secure software development, and continuous monitoring.

Security is integrated into every stage of the software lifecycle, from design and development to deployment and ongoing operations.

Security Philosophy

Work-A-Beez follows the principle of Security by Design.

Every new feature is evaluated for:

  • Confidentiality
  • Integrity
  • Availability
  • Privacy
  • Authentication
  • Authorization
  • Auditability
  • Risk

Security reviews occur throughout the software development lifecycle.

Shared Responsibility Model

Security is a shared responsibility between Work-A-Beez and our customers.

Work-A-Beez Responsibilities

We are responsible for:

  • Application security
  • Infrastructure security
  • Database protection
  • Software updates
  • Monitoring
  • Data backups
  • Service availability
  • Vulnerability remediation

Customer Responsibilities

Customers are responsible for:

  • Protecting user credentials
  • Managing employee access
  • Assigning permissions appropriately
  • Enabling Multi-Factor Authentication
  • Maintaining secure endpoint devices
  • Reviewing audit logs
  • Promptly removing inactive users

Cloud Infrastructure

Work-A-Beez is hosted within professionally managed cloud infrastructure designed for high availability and security.

Infrastructure includes:

  • Redundant systems
  • Secure networking
  • Firewall protection
  • Geographic redundancy
  • Automated monitoring
  • Encrypted storage
  • Secure backup systems

Infrastructure providers maintain their own physical security controls and environmental protections.

Data Encryption

Data In Transit

All communication between users and Work-A-Beez is encrypted using modern TLS (Transport Layer Security).

This protects information while traveling across public networks.

Examples include:

  • Login requests
  • Employee records
  • Payroll information
  • API traffic
  • Administrative actions

Data At Rest

Sensitive customer data is encrypted while stored within Work-A-Beez systems.

Examples include:

  • Employee information
  • Payroll records
  • Timecards
  • Company settings
  • Uploaded documents

Authentication

Every user must authenticate before accessing the platform.

Authentication includes:

  • Email address
  • Username (optional)
  • Password
  • Multi-Factor Authentication (when enabled)

Unauthorized access attempts are monitored and logged.

Authorization

Access to platform resources is controlled through Role-Based Access Control (RBAC).

Examples of supported roles include:

  • System Administrator
  • Company Administrator
  • Human Resources
  • Payroll Administrator
  • Manager
  • Supervisor
  • Employee
  • Auditor

Permissions are granted based on the principle of least privilege.

Users receive only the permissions necessary to perform their responsibilities.

Multi-Factor Authentication

Work-A-Beez supports Multi-Factor Authentication (MFA) for enhanced account security.

When enabled, users must provide:

  • Password
  • One-time verification code

MFA significantly reduces the risk of unauthorized account access resulting from compromised passwords.

Organizations are encouraged to require MFA for all administrative users.

Password Security

Passwords are never stored in plain text.

Security practices include:

  • Secure password hashing
  • Password complexity requirements
  • Minimum password length
  • Password change capabilities
  • Password reset verification
  • Failed login monitoring

See the Password Policy for complete requirements.

Session Security

User sessions are protected through multiple security mechanisms.

These include:

  • Secure session tokens
  • Automatic session expiration
  • Logout capability
  • Session invalidation after password changes
  • Idle timeout

Users should always sign out when using shared devices.

Secure Software Development

Security is integrated throughout the development lifecycle.

Development practices include:

  • Code reviews
  • Peer review
  • Static code analysis
  • Dependency review
  • Secure coding guidelines
  • Automated testing
  • Security testing
  • Regression testing

Security considerations are evaluated before every production release.

Application Security

Application-level protections include:

  • Input validation
  • Output encoding
  • Access control enforcement
  • CSRF protection
  • Rate limiting
  • Session validation
  • Error handling
  • Secure file handling

Security controls are regularly reviewed and updated.

Infrastructure Security

Infrastructure protections include:

  • Network segmentation
  • Firewall management
  • Secure administrative access
  • Monitoring
  • Intrusion detection
  • Secure backups
  • Patch management

Only authorized personnel have administrative access.

Network Security

Network protections include:

  • Firewalls
  • Secure routing
  • TLS encryption
  • DDoS mitigation (where supported)
  • Traffic monitoring
  • Access restrictions

Logging & Monitoring

Work-A-Beez continuously monitors platform activity.

Examples include:

  • Authentication events
  • Administrative actions
  • System errors
  • Permission changes
  • API requests
  • Infrastructure health

Logs help identify:

  • Unauthorized access attempts
  • System anomalies
  • Operational issues
  • Potential security incidents

Audit Logging

Administrative activities may be recorded for security and compliance purposes.

Examples include:

  • User creation
  • User deletion
  • Password resets
  • Permission modifications
  • Timecard edits
  • Schedule changes
  • PTO approvals
  • Billing updates

Audit records assist organizations with compliance and internal investigations.

Vulnerability Management

Work-A-Beez actively monitors for security vulnerabilities.

Processes include:

  • Software dependency reviews
  • Security updates
  • Risk assessments
  • Vulnerability remediation
  • Security testing

Critical vulnerabilities are prioritized for immediate remediation.

Security Updates

Security improvements are deployed regularly.

Updates may include:

  • Vulnerability patches
  • Dependency updates
  • Performance improvements
  • Infrastructure enhancements
  • Security enhancements

Maintenance windows may be announced when customer action is required.

Backup & Recovery

Customer data is backed up regularly to support disaster recovery objectives.

Backup procedures are designed to protect against:

  • Hardware failures
  • Software failures
  • Operational errors
  • Infrastructure disruptions

Backup retention periods may vary.

Disaster Recovery

Work-A-Beez maintains disaster recovery procedures intended to restore services following major operational disruptions.

Recovery priorities include:

  • Platform availability
  • Data integrity
  • Secure restoration
  • Service continuity

Specific recovery objectives may vary by subscription level.

Incident Response

Security incidents are handled using a structured response process.

Typical phases include:

  1. Detection
  2. Investigation
  3. Containment
  4. Remediation
  5. Recovery
  6. Post-Incident Review

Customers may be notified when required by applicable law or contractual obligations.

Responsible Disclosure

We appreciate responsible reporting of security vulnerabilities.

Researchers who discover potential vulnerabilities should:

  • Report findings privately
  • Allow reasonable time for investigation
  • Avoid disrupting customer services
  • Avoid accessing customer information

See our Responsible Disclosure Policy for complete reporting procedures.

Customer Security Best Practices

We recommend customers:

  • Enable MFA
  • Use unique passwords
  • Remove inactive accounts
  • Review permissions regularly
  • Monitor audit logs
  • Keep browsers updated
  • Use supported operating systems
  • Train employees on phishing awareness

Security is strongest when customers actively participate in protecting their environments.
