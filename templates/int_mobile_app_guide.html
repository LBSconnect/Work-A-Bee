Welcome to the Work-A-Beez Mobile Application Guide.

The Work-A-Beez mobile application enables employees and managers to stay connected to their workforce from anywhere. Users can clock in and out, manage schedules, request leave, receive notifications, approve requests, and monitor workforce activity using a secure mobile experience.

This guide explains installation, configuration, security, and everyday mobile workflows.

Introduction

The Work-A-Beez mobile application allows authorized users to perform workforce management tasks from their smartphones or tablets.

Depending on assigned permissions, users may:

  • View schedules
  • Clock in and out
  • Start and end breaks
  • Request leave
  • View PTO balances
  • Receive announcements
  • Approve requests
  • Manage employees
  • Receive push notifications

Available features depend on the user's security role.

System Requirements

Supported platforms include:

iOS

  • Current supported iOS versions
  • Internet connection
  • Location Services (for GPS features)

Android

  • Current supported Android versions
  • Internet connection
  • Location Services (for GPS features)

Recommended

  • Automatic app updates enabled
  • Push notifications enabled
  • Biometric authentication (if supported)
  • Device screen lock configured

Installing the Mobile App

Download the application from the appropriate app store.

After installation:

  1. Open the application.
  2. Enter your organization information.
  3. Sign in using your company credentials.
  4. Complete Multi-Factor Authentication if required.
  5. Grant required permissions.

Signing In

Users authenticate using organization credentials.

Supported methods include:

  • Email and password
  • Single Sign-On (SSO)
  • Microsoft Entra ID
  • Google Workspace
  • OAuth providers
  • SAML identity providers

Authentication methods depend on organizational configuration.

Multi-Factor Authentication (MFA)

If enabled, users may verify their identity using:

  • Authenticator application
  • SMS verification code
  • Email verification
  • Security key (where supported)

MFA significantly improves account security and may be required before accessing sensitive features.

Mobile Dashboard

After signing in, the dashboard displays personalized information such as:

  • Today's schedule
  • Upcoming shifts
  • Current clock status
  • PTO balance
  • Pending approvals (managers)
  • Recent notifications
  • Company announcements
  • Quick action buttons

Dashboard content varies based on the user's role.

Clocking In & Out

Employees can record work hours directly from the app.

Typical workflow:

  1. Open the app.
  2. Select Clock In.
  3. Confirm work location if prompted.
  4. Begin the shift.

To end the workday:

  1. Select Clock Out.
  2. Review worked hours.
  3. Confirm submission.

Organizations may require GPS verification or geofencing before allowing clock-in.

Breaks & Meal Periods

Employees can manage breaks by selecting:

  • Start Break
  • End Break
  • Start Meal
  • End Meal

Break rules are determined by organizational policy.

Missed or incomplete breaks may require supervisor review.

Viewing Schedules

Employees can view:

  • Today's schedule
  • Weekly schedule
  • Monthly schedule
  • Assigned shifts
  • Open shifts
  • Approved leave
  • Holiday schedules

Schedules automatically update after publication.

Shift Marketplace

If enabled by the organization, employees may:

  • View open shifts
  • Volunteer for available shifts
  • Request shift swaps
  • Offer assigned shifts
  • Withdraw swap requests

Managers receive notifications when approval is required.

Leave Requests

Employees can:

  • Submit leave requests
  • View PTO balances
  • Check approval status
  • Cancel pending requests
  • Review leave history

Leave requests follow the organization's configured approval workflow.

Manager Functions

Managers may access additional capabilities including:

  • Approve timecards
  • Approve leave requests
  • View employee schedules
  • Fill open shifts
  • Monitor attendance
  • Review missed punches
  • Send team announcements

Manager features are available only to authorized users.

Notifications

The mobile app supports push notifications for events such as:

  • Upcoming shifts
  • Schedule changes
  • Leave approvals
  • Missed punch reminders
  • Payroll reminders
  • Company announcements
  • Approval requests

Users may manage notification preferences within the application settings.

Offline Mode

Limited functionality may be available when internet connectivity is unavailable.

Offline capabilities may include:

  • Viewing cached schedules
  • Reviewing previously downloaded information

Actions requiring server validation, such as clocking in or submitting leave requests, may be queued and synchronized automatically when connectivity is restored.

Administrators should define organizational policies regarding offline time capture.

Security & Privacy

To protect user information, the mobile application supports:

  • Encrypted communication
  • Secure token storage
  • Automatic session expiration
  • Biometric authentication
  • Remote session invalidation
  • Multi-Factor Authentication

Users should report lost or stolen devices immediately.

Device Management

Administrators may enforce mobile security policies such as:

  • Minimum operating system version
  • Screen lock requirements
  • Device encryption
  • Session timeout
  • Application version requirements

Organizations using Mobile Device Management (MDM) solutions may apply additional security controls.

Troubleshooting

Unable to Sign In

Verify:

  • Username and password
  • Internet connection
  • Organization URL
  • Multi-Factor Authentication status

GPS Not Detected

Check:

  • Location Services enabled
  • App permissions granted
  • Device GPS enabled
  • Organizational geofencing rules

Push Notifications Not Received

Verify:

  • Notifications enabled on the device
  • App notification permissions
  • Internet connectivity
  • User notification preferences

Unable to Clock In

Confirm:

  • Scheduled shift exists
  • Location requirements met
  • Device date and time are correct
  • Organization policy permits mobile clock-in

Frequently Asked Questions

Can I use multiple devices?

Yes, if permitted by your organization. Active sessions may be limited based on security policies.

What happens if I lose internet connectivity?

Certain information remains available from the local cache. Actions requiring server confirmation are synchronized once connectivity returns.

Can I disable location services?

Location services may be optional or required depending on your organization's attendance policies. If required, disabling location services may prevent mobile clock-in.

How do I change my notification preferences?

Open Settings → Notifications within the mobile application to update your preferences, subject to organizational policies.
